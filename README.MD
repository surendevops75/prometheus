This repository contains documentation and configuration files for setting up a complete monitoring and alerting solution using:

- Prometheus
- Node Exporter

The project demonstrates how to collect infrastructure metrics, monitor system health, create alerts, and manage notifications in a production-like environment.

## 🔹 Components

### Prometheus

Prometheus is an open-source monitoring and alerting toolkit.

#### Responsibilities

- Collect metrics from targets
- Store time-series data
- Execute PromQL queries
- Evaluate alerting rules
- Integrate with Alertmanager

---

### Node Exporter

Node Exporter exposes Linux system metrics for Prometheus.

#### Metrics Collected

- CPU Usage
- Memory Utilization
- Disk Usage
- Network Statistics
- System Load
- Filesystem Information

---

### Alertmanager

Alertmanager handles alerts generated by Prometheus.

#### Responsibilities

- Receive alerts
- Group alerts
- Silence alerts
- Route notifications
- Send Email Notifications

---

## 🔄 Monitoring Workflow

1. Node Exporter exposes system metrics.
2. Prometheus scrapes metrics at configured intervals.
3. Prometheus stores metrics in its time-series database.
4. Alert rules are continuously evaluated.
5. Alertmanager receives triggered alerts.
6. Notifications are sent to configured receivers.
